# Voxtype Fork Release `v0.6.6-kde.1`

This first fork release adds an optional KDE Plasma Wayland listening overlay on top of upstream Voxtype `v0.6.6`.

## Highlights

- adds `extras/kde-listening-overlay`, a separate helper executable that watches Voxtype's runtime state file
- shows a slim recording HUD above the KDE Plasma panel while Voxtype is actively recording
- keeps core Voxtype behavior unchanged for users who do not want the overlay
- includes sample config, systemd user units, KDE launcher examples, and implementation notes

## Scope

- tested on Arch Linux with KDE Plasma 6 Wayland
- depends on Qt6 Widgets and LayerShellQt
- currently tuned for a bottom-right Plasma panel layout

## Notes

This is a KDE-specific UX enhancement for toggle-mode workflows. It is not a patch to Voxtype core, and it remains optional inside the fork.
